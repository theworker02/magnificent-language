# GitHub Release Guide

This guide covers releasing both the MGL runtime and the bundled VS Code extension.

## Release Checklist

1. Confirm the working tree is clean.
2. Run:

```bash
npm install
npm test
```

3. Bump the version in:

- `package.json`
- `mgl-extension/package.json`
- any README badges or release notes that mention the version explicitly

4. Review the extension icon and README files if the release changes branding or editor features.

## Packaging The VS Code Extension

From the repository root:

```bash
cd mgl-extension
npx @vscode/vsce package
```

This should produce a file named like:

```text
magnificent-language-vscode-1.0.0.vsix
```

Do not commit the generated `.vsix` file. Attach it to the GitHub release instead.

## Tagging And Publishing

From the repository root:

```bash
git add .
git commit -m "Release v1.0.0"
git tag v1.0.0
git push origin main --tags
```

Then create a GitHub release for `v1.0.0` and attach:

- the packaged `.vsix` from `mgl-extension/`
- optional source archives generated by GitHub

## Recommended Release Notes

Include:

- language/runtime changes
- CLI changes
- REPL or module-system updates
- VS Code extension changes
- any breaking changes or migration notes

## Suggested Asset Policy

- Keep generated release artifacts out of git.
- Store icons, screenshots, and committed brand assets in source control.
- Upload packaged binaries like `.vsix` files to GitHub Releases.
